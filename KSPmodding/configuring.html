Tips

It's best reference a part that closely matches yours, especially if you are starting out, it is perfectly fine as long as you don't copy everything exactly word for word.

Keep your config neat, maybe even commented, just so you and other modders can understand your code and possibly modify it in the future. You can write comments using a double slash (//).

While you can edit configs using Notepad, I personally prefer to use Visual Studio Code with KSP CFG Support addon as it is easier to see what i'm doing. This is completely up to you.

Consider reading the KSP CFG File Documentation, though it is a little outdated.

Basics

Let's start with the name, module and author. This is simple. I personally like to name my part PROXI-[part name] just so that it is easier to call it from other configs. For module, you can leave it as Part, i'm assuming you are making a part. Finally for author, write your alias.

Now we need the model. Use MODEL{ model = yourModName/yourPartDirectory } to get the model in to the config.

Assuming you created the nodes in Blender (see here) use NODE{}, within it should be name, transform, size, and method. name is the... name of your node, transform is the name of the node in Unity (should be node_[something] ), size is how large the attachment is, most of the time is either size 2 or 3 in my experience, finally method is how the attachment would work, most of the time FIXED_JOINT does the trick.

Afterwards you have the rest of the basic configuration, I won't go through them all but here are the important ones.

  • attachRules = 1,1,1,1,1 ; with the tags being: stack, srfAttach, allowStack, allowSrfAttach, allowCollision
  • maximum_drag and minimum_drag should both be left as 0.1 unless it's a wing or you know what you're doing
  • maxTemp - max temperature of the entire part
  • skinMaxTemp - max temperature of the surface of the part

Modules

ModuleAnimateGeneric

  • animationName = animation name in Unity
  • startEventGUIName = event name before animation plays
  • endEventGUIName = event name after animation finishes
  • actionGUIName = name of event in action group editor
  • allowDeployLimit = limit the length of animation
  • revClampDirection = leave as false
  • revClampSpeed = whether animation can reverse
  • revClampPercent = leave as true
  • toggleAction = whether action can be toggled
  • defaultActionGroup = as the name suggests
