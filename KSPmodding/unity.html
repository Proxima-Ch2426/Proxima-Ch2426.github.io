Unity

Exporting from Blender

Before exporting, ensure that your part is centered in world space and orientated as if it is in the VAB.

Select the object you wish to export, the go to File > Export > FBX.

In the popup window, set Limit to Selected Objects. Object Types should be set to Empty, Armature, Mesh and Other. Ensure Apply Transform is unchecked, it never works for me.

If you have animations, make sure the Animation check box is checked, and disable all but Key All Bones. KSP only supports one animation per part.

Importing to Unity

Drag and drop your FBX file into your Unity project, there are a couple of things we need to do before working on it.

Click on your imported FBX in the file browser, in the Inspector, go to Materials and Extract Materials (assuming you aren't reusing textures). Your materials should be in the file browser now.

If your part has animations, go to Rig and set Animation Type to Legacy. Then go to Animation and change the name to something easier to remember, then hit apply.

Your part should now be ready.

Materials

We will be creating a material with an emissive texture. Under the extracted materials, find the one you want to edit. Under Shader, select KSP/Emissive/Bumped Specular (it would be KSP/Bumped Specular for regular materials). Drag and drop the correct textures into the respective maps.

You might get a warning saying the bump map is not actually a bump map... well it is, and there is likely a Fix Now button under the texture box, just click that and the issue is resolved.

Also for some reason _Color is not always set to pure white, so make sure it is, otherwise your parts might appear darker than they actually are.

Creating engine cores

This was referenced from The KSP Part Modding Guide

The above image, created by Nertea, shows how to create and configure an engine emissive with heat and throttle glow. A separate mesh is required on the inside of the bell.

This mesh should have its own material using the KSP/Particles/Additive shader.

Its transform name should be unique for use in the config file (it is named STBEEngineCore in the image).

The Unity Layer should be set to TransparentFX, with the Unity Tag set to Icon_Hidden so that the glow does not show in the part selection icons.

Hide the emissive by unchecking the check box in the Inspector before exporting.

For ModuleDepthMask users: Note that if you intend to use depth masks later on, your engine core will not appear due to rendering limitations. A work around is to compress your engine core mesh on the vertical axis until it is nearly flat, and move it slightly out of the depth mask mesh. (Thanks NoLifeJordan)

Colliders

Select your collider meshes, remove the Mesh Renderer and add the Mesh Collider component, ensure that convex is checked. Reminder that KSP does not support concave colliders.

Exporting from Unity

Assuming you have followed PartTool's installation instructions, the following should be a breeze. In the main parent transform, add the Part Tools component. Fill up the model's name and set the file URL to your mod. If you are writing textures for the first time, leave the Write Textures check box ticked. Once you are ready click Write.

Note that PartTools export textures as PNGs. KSP has a terrible PNG convertor, so it is recommended to use DDS files. See here for how to export DDS files from Gimp.

Unity layers

Not all unity layers will be mentioned here, just those that are important for part modding.

  • Layer 1 - TransparentFX
  • Layer 2 - Ignore Raycast
  • Layer 16 - Kerbals - For items supposed to be visible in IVA views.
  • Layer 20 - Internal Space - Prevent IVA views from being blocked by props or hull parts, used when creating internal props.
  • Layer 21 - Part Triggers - For hatches and ladders together with box colliders with "is Trigger" checkbox, tag should be placed on this collider.
  • Layer 23 - AeroFXIgnore - Prevent part from showing aero effects
  • Layer 26 - WheelCollidersIgnore - Prevent unwanted interactions between wheel colliders and other colliders.
  • Layer 27 - WheelColliders - For wheel colliders
