Rigging

Gimbal links

Most engines use hydraulic pistons for gimbal control, with the upper half connecting to the vehicle's body, and the lower half connecting to the engine. This can be achieved using the FXModuleLookAtConstraint module.

Alternatively you can use ModuleAdvancedLookAtConstraint from the DeployableEngines mod, which handles rotation much better and prevents the strange rolling issue you can sometimes see.

Create two empties, preferably using the Arrows Empty. Place them at where you want the pivot points of your links to be, and use the Track To Object Constraint with the Track Axis set to Z with Y as up, then apply the constraint. Do this for both empties. Model your links and ensure that you copy the empty's rotation to the model. Then parent your models to the empties. It should look something like this:

Deforming meshes

This is especially useful for things like tubing or engine mounts that require to be deformed as the engine gimbals. This requires the use of an armature.

In your armature, create two bones, we will name it fixed and gimbal. Just like the gimbal links, ensure the bones' Z axis are facing each other. To check the axis, go to the armature's Object Data Properties and enable "Axes" under Viewport Display. You might also want to change the bone display to Stick for easier viewing.

Create two vertex groups in the mesh you want to deform, with the names being fixed and gimbal, and weight paint the respective vertex groups. Once done, parent the mesh to the armature using Armature Deform.

Once you checked your mesh is deforming correctly, parent the rest of the gimballing engine to the armature using Bone, make sure gimbal is selected in Pose Mode.

Landing gears (using KSPWheel)

Landing gears have got to be the most difficult thing to configure, but I will try my best to keep things simple. I will be rigging using the KSPWheel mod's way as it simplfies things a lot more than the stock system in my opinion, plus it is much easier to configure later on using KSPWheel, this however, would require KSPWheel to be a hard dependency for your mod. Also it's good to have a reference if you are becoming lost.

Create your main landing gear strut mesh, we will name it WheelRetract, this part will have the extending and retracting animation.

Next, create an empty called deployTgt which would be located at the center of the tire.

Afterwards, create two empties called SteeringActuator and SteeringPivot, with the first located at the end of the main strut, and the latter at the center of the tire.

Create an empty called WheelCollider, this would be the resting position of the wheel when weight is applied on the gear.

Almost there... create two more empties called SuspensionPivot and SuspensionPivotNeutral, the first controls the up and down movement of the wheel, with the latter being the rest position of the wheel when it is hanging. Both empties have the same origins.
